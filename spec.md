# 点眼記録アプリ 開発仕様書

**バージョン:** 1.0.0
**最終更新:** 2026-03-16
**対象ファイル:** `index.html`

---

## 1. 概要

点眼（目薬）の実施記録をつけ、継続率を可視化するシングルページWebアプリ。
サーバー不要。ブラウザのみで動作し、iPhoneのホーム画面に追加してアプリとして利用できる。

---

## 2. 技術スタック

| 種別 | 内容 |
|------|------|
| 構成 | シングルHTMLファイル（`index.html`） |
| スタイル | インラインCSS（モバイルファースト） |
| スクリプト | バニラJavaScript（ES6+） |
| グラフ | Chart.js 4.4.0（CDN） |
| データ永続化 | `localStorage`（キー: `tengan_v1`） |
| 音声入力 | Web Speech API（`webkitSpeechRecognition`） |

### 動作環境

| 環境 | 要件 |
|------|------|
| プライマリ | iPhone / iOS Safari 14.5以上 |
| セカンダリ | Android Chrome、デスクトップ Chrome/Edge/Safari |
| オフライン | 対応（CDNを除く） |
| PWA | `apple-mobile-web-app-capable` 対応（ホーム画面追加でスタンドアロン動作） |

## 3. データ設計

### ストレージ

- **キー:** `tengan_v1`
- **形式:** JSON文字列
- **場所:** `localStorage`

### データ構造

```json
{
  "2026-01-05": true,
  "2026-01-06": true,
  "2026-03-16": true
}
```

- キー: `YYYY-MM-DD` 形式の日付文字列
- 値: 常に `true`（点眼した日のみ記録。点眼しなかった日のキーは存在しない）

### CSVフォーマット

エクスポート・インポートで使用するCSVの仕様。

| 項目 | 内容 |
|------|------|
| 文字コード | UTF-8（BOM付き） |
| 改行コード | CRLF または LF |
| ヘッダー | 1行目（必須） |
| ファイル名 | `tengan_records_YYYY-MM-DD.csv` |

**列定義:**

| 列名 | 型 | 説明 |
|------|----|------|
| `日付` | 文字列 | `YYYY-MM-DD` 形式（必須） |
| `点眼` | 数値 | `1` 固定（エクスポート時）。インポート時は無視 |

**サンプル:**

```
日付,点眼
2026-01-05,1
2026-01-06,1
2026-03-16,1
```

## 4. 機能仕様

### 4.1 点眼記録

| 項目 | 内容 |
|------|------|
| トリガー | 「点眼を記録する」ボタン押下、または音声コマンド「点眼」 |
| 動作 | 当日の日付をストレージに保存 |
| 重複 | 同日に複数回実行しても1回として記録（上書きなし） |
| フィードバック | トーストメッセージ表示・画面全体を即時更新 |

### 4.2 点眼率表示

| 区分 | 計算式 |
|------|--------|
| 当月 | 当月の点眼日数 ÷ 月初から今日までの日数 |
| 前月 | 前月の点眼日数 ÷ 前月の総日数 |

- 結果は整数パーセント（`Math.round`）で表示
- 色分け: 80%以上→緑、50%以上→黄、50%未満→赤

### 4.3 グラフ表示

| 項目 | 内容 |
|------|------|
| 種類 | 折れ線グラフ（Chart.js） |
| 期間 | 直近6ヶ月（当月含む） |
| 軸 | 横軸: 月名（例: `3月`）、縦軸: 点眼率（0〜100%） |
| 当月の扱い | 月初から今日までの日数で計算 |
| 表示 | 「直近6ヶ月グラフ」セクションをタップして展開・折りたたみ |

### 4.4 今日の記録削除

| 項目 | 内容 |
|------|------|
| トリガー | 「今日を削除」ボタン、または音声コマンド「削除」 |
| 動作 | 確認ダイアログ表示後、当日のキーをストレージから削除 |
| 未記録時 | 「今日の記録はありません」トーストを表示（削除ダイアログは出ない） |

### 4.5 全記録削除

| 項目 | 内容 |
|------|------|
| トリガー | 「全削除」ボタン |
| 動作 | 確認ダイアログ（件数表示）後、ストレージを空オブジェクトで上書き |
| 記録なし時 | 「記録がありません」トーストを表示 |

### 4.6 CSVエクスポート

| 項目 | 内容 |
|------|------|
| トリガー | 「エクスポート」ボタン、または音声コマンド「エクスポート」「ダウンロード」 |
| 出力 | UTF-8 BOM付きCSVファイルをダウンロード |
| 記録なし時 | 「エクスポートするデータがありません」トーストを表示 |

### 4.7 CSVインポート

| 項目 | 内容 |
|------|------|
| トリガー | 「インポート」ボタン |
| 入力 | `.csv` ファイル選択ダイアログ（`accept=".csv,text/csv"`） |
| 文字コード | UTF-8（BOM自動除去） |
| 列特定 | ヘッダー行の `日付` 列を検索（列順不問） |
| マージ方針 | 既存記録を保持し、新規レコードのみ追加（上書きしない） |
| 確認 | インポート件数・スキップ件数を確認ダイアログで提示後に実行 |

**バリデーション:**

| 状態 | 処理 |
|------|------|
| `日付` 列が存在しない | エラートースト表示、インポート中止 |
| データ行が0行 | エラートースト表示、インポート中止 |
| `YYYY-MM-DD` 形式でない行 | その行をスキップ（無効行カウントに加算） |
| 無効な日付値（例: `2026-13-01`） | その行をスキップ |
| すでに記録済みの日付 | スキップ（既存件数カウントに加算） |

### 4.8 音声入力

| 発話 | 動作 |
|------|------|
| 「点眼した」「点眼」 | 今日を記録 |
| 「グラフ」「チャート」「ぐらふ」 | グラフを展開してスクロール |
| 「削除」「さくじょ」 | 今日の記録削除（確認ダイアログあり） |
| 「エクスポート」「ダウンロード」「えくすぽーと」 | CSVエクスポート |
| 上記以外 | 「コマンドが認識できませんでした」と表示 |

- 言語設定: `ja-JP`
- 非対応ブラウザではマイクボタンを無効化し、メッセージを表示

## 5. UI仕様

### 画面構成（上から順）

1. **ヘッダー** — アプリ名・今日の日付（曜日付き）
2. **今日の記録カード** — ステータス表示・記録ボタン・マイクボタン・音声ログ
3. **点眼率カード** — 当月・前月の点眼率（数値＋詳細）
4. **グラフカード** — 折りたたみ式・Chart.jsによる折れ線グラフ
5. **今月の記録一覧** — 折りたたみ式・新しい日付順
6. **データ管理カード** — エクスポート・インポート・削除ボタン
7. **音声コマンド一覧** — ヘルプ表示

### デザイン原則

- タッチターゲット: 44px以上
- プライマリカラー: `#2563eb`
- フォント: `-apple-system, BlinkMacSystemFont, 'Hiragino Sans'`
- 角丸: 12px
- 通知: トースト（下部中央、2.5秒後自動消去）

### 確認ダイアログ

- 削除系: OKボタンは赤（`btn-danger`）、ラベル「削除する」
- インポート: OKボタンは青（`btn-primary`）、ラベル「N件をインポート」

## 7. 既知の制限

| 項目 | 内容 |
|------|------|
| データ同期 | 複数端末間の同期なし（ローカル保存のみ） |
| バックアップ | 手動でCSVエクスポートが必要 |
| 音声認識 | iOS Safariはオフライン時に音声認識が利用不可 |
| ダウンロード | iOSではCSVが「ファイル」アプリに保存される（自動ダウンロードフォルダなし） |
| CDN依存 | Chart.jsはCDN読み込みのためオフライン時グラフ非表示 |
| ストレージ上限 | `localStorage` は通常5MB（数十年分の記録は十分格納可能） |
